## Local and LAN development (Planted)

- **Next.js 16** in this repo. Dev server may use **Turbopack** (`next dev`).
- **API client** (`src/lib/api.ts`): In development, prefer **relative** `/api/...` (empty base URL) so the browser stays on the Next origin. **`next.config.ts`** rewrites `/api/:path*` and `/images/:path*` to Spring at **`PLANTED_API_PROXY_ORIGIN`** (default `http://127.0.0.1:8080`). Do not rely on `localhost:8080` in the browser on other devices.
- **`allowedDevOrigins`:** Populated from the machine’s private IPv4 addresses so opening the app via `http://192.168.x.x:3000` works. Extend with **`PLANTED_DEV_EXTRA_ORIGINS`** if needed.
- **Images:** DTOs may use root-relative `/images/...`; **`PlantImageStrip`** must treat **`/`** URLs like app-hosted images (not only `http:`).
- **Shipping:** If the user says **ship**, update **README** / this file when conventions change, and follow **`scripts/ship-checklist.md`** + **`.cursor/rules/ship.mdc`**.
- **Plant detail types:** `PlantDetailResponse` may include **`historyDailyDigests`** (preferred History tiles in About when non-empty); align `src/types/plant.ts` with the backend contract.
- **Plant detail journal UI:** Text and optional photo for observations live at the **bottom of the Care** panel. Text-only notes use `addHistoryNote`; submitting a **photo** uses `uploadPlantPhoto` (`POST /api/plants/{id}/photos`) which promotes the upload to the hero/main image, pushes the prior photo into the left-column Photo history strip, auto-records a history entry, and refreshes the health bio section. The About **History** block shows the merged timeline (digests, summary, and structured `historyEntries`).
- **Care topics:** On `/plants/[id]`, watering through placement use `CareTopicAccordion` (exclusive expand, smooth height transition). Collapsed due rows show a Lucide warning icon only when `wateringDue` / `wateringOverdue`, `fertilizerDue`, `pruningDue`, `lightAttentionNeeded`, or `placementAttentionNeeded` is true, not duplicated primary text in the header. Wire `actionNeeded` on the Light and Placement accordions from `reminderState.lightAttentionNeeded` / `reminderState.placementAttentionNeeded` — those flags are denormalised from the corresponding bio section's `attentionNeeded` on the backend.
- **`ReminderIconRow`:** Always renders six fixed icons (`Droplets`, `Leaf`, `Scissors`, `Sun`, `MapPin`, `HeartPulse`) on plant list cards and `/screensaver` tiles. Illumination is derived from `ReminderStateDto` via the pure helper **`src/components/plant/reminderIconState.ts#computeIconState`** — keep all colour/label logic there (unit-tested in `reminderIconState.test.ts`) so the component stays a thin render. `*AttentionReason` strings become the aria-label / tooltip when the matching flag is set.
- **Placement seed:** `updatePlantPlacement` → **`PATCH /api/plants/{id}/placement`** (202 + same body shape as reanalysis). The Care panel edits `location` via the Placement pencil dialog; do not show raw seed as duplicate body copy next to `placementGuidance`.
- **TanStack Query Devtools:** Opt in with **`NEXT_PUBLIC_QUERY_DEVTOOLS=true`** in `.env.local` (development only); see `src/lib/providers.tsx`.
- **Plant bio sections (decomposed cache):** `PlantDetailResponse.bioSections` is a `Partial<Record<BioSectionKey, BioSectionDto>>` keyed by `SPECIES_ID`, `HEALTH_ASSESSMENT`, `SPECIES_DESCRIPTION`, `WATER_CARE`, `FERTILIZER_CARE`, `PRUNING_CARE`, `LIGHT_CARE`, `PLACEMENT_CARE`, `HISTORY_SUMMARY`. Always read sections through the helpers in **`src/lib/bioSections.ts`** (`get*Section`, `flattenBioSectionsToAnalysis`, `hasRefreshingBioSections`). On `/plants/[id]` the page polls while `hasRefreshingBioSections(data)` is true and uses `flattenBioSectionsToAnalysis(plant)` as the `analysis` value so legacy UI keeps working during backfill. Pass a `refreshing` prop to `CareTopicAccordion` (and the species/health panels) when the corresponding section's `isRefreshing` is true; render a subtle "Updating…" hint, not a blocking spinner.
- **Shared bio layout:** The two-column plant bio (hero + thumbs on the left, header/status/species/care/history on the right) lives in **`src/components/plant/PlantBioView.tsx`**. `/plants/[id]` renders it with mutations wired; `/screensaver` renders it with **`readOnly`** to hide every interactive control (name/growing/placement pencils, placement & water dialogs, watered/fertilized/pruned + refresh/archive rows, care observation input, and the history summary button). Keep new bio UI inside `PlantBioView` so both surfaces stay in sync, and gate anything interactive on the `readOnly` prop.
